01 Contexte & objectifs

1.1 Contexte

GPE Énergies & Services (Vitrolles, depuis 2008) intervient sur cinq métiers complémentaires : électricité courants forts/faibles, énergies renouvelables, CVC, VRD et bureau d'études. La cible principale est constituée de maîtres d'ouvrage publics et privés, promoteurs, bailleurs sociaux, collectivités et entreprises tertiaires.

1.2 Objectifs business

  • Présenter les 8 expertises métier de manière claire et hiérarchisée.
  • Convertir les visiteurs en leads qualifiés via un formulaire direct.
  • Crédibiliser par références clients, certifications et chiffres clés.
  • Référencer sur les requêtes métier (SEO local + sectoriel).

1.3 Personas cibles

PersonaBesoin principal
Maître d'ouvrage / promoteurComprendre le périmètre, demander un devis
Bureau d'études externeVérifier les compétences, identifier les certifications
CollectivitéConfirmer les références similaires, contacter
Particulier (rénovation)Trouver le service adapté, obtenir un contact

1.4 Critères de succès

  • Score Lighthouse Performance ≥ 90 sur mobile.
  • Formulaire de contact opérationnel avec envoi email automatique.
  • Site 100% responsive (mobile, tablette, desktop, grand écran).
  • Délai d'affichage initial < 2 s sur 4G.

03 Fonctionnalités détaillées

3.1 Navigation globale (Navbar)

  • Position : fixe en haut, persistante sur toutes les pages.
  • Comportement : verre transparent au repos, blur intensifié + bordure fine au scroll (> 24 px).
  • Contenu : logo + nom, 4 liens d'ancre (Services / Réalisations / À propos / Contact), téléphone cliquable, CTA pilule sombre « Demander un devis ».
  • Effets : soulignement animé au survol (origin-left scale-x).
  • Mobile : burger 3 traits fins, menu plein écran avec scroll body verrouillé, flèches discrètes par item.

3.2 Pied de page (Footer)

Fond noir, 4 colonnes en desktop, repliées en colonne unique en mobile :

  1. Marque : logo + baseline + icônes sociales.
  2. Navigation : 5 liens principaux.
  3. Services clés : 5 services (choix éditorial, pas la liste exhaustive).
  4. Contact : téléphone + email + adresse + indicateur « réponse 24 h ouvrées ».

Barre du bas : copyright et 3 liens légaux.

3.3 Hero d'accueil

  • Section plein écran (min-h-screen) avec gradient pastel et 2 cercles flous diffus.
  • Eyebrow sobre « Vitrolles · depuis 2008 ».
  • Titre principal Playfair Display — taille fluide 4xl mobile → 8xl desktop, segment « donne vie » mis en surbrillance bleue animée.
  • 2 CTAs : Contactez-nous (primaire) + Nos Services (secondaire).
  • 4 chiffres clés avec compteurs animés (0 → cible) déclenchés à l'apparition dans la vue, easing cubique sur 1,6 s, joués une seule fois.

3.4 Section Services

  • Grille adaptative 1 / 2 / 3 / 4 colonnes.
  • Carte = image (16:9), badge couleur + icône SVG, titre, description courte (clamp 3 lignes), lien En savoir plus.
  • Animations fade-up séquentielles (stagger 80 ms).

3.5 Section Réalisations

  • Grille de logos clients (2/5 cols) — 10 placeholders typographiques.
  • 6 cartes projets de référence (label + localisation + tag de service).

3.6 Section À propos

  • Fond noir (rupture visuelle), titre + paragraphes, badges certifications RGE QualiPV, Qualifelec, RGE QualiClimat, IRVE Qualif., Qualibat.
  • Colonne droite : 3 valeurs + bloc « snapshot » 3 colonnes (2008 · Création / +40 · Collaborateurs / Sud · Région).

3.7 Section Contact

Colonne informations

Titre, sous-titre, 3 contacts cliquables (tel / mailto / Maps), encart horaires.

Colonne formulaire

Nom, Email, Téléphone, Société, Service (select), Message + honeypot caché. États idle → loading → success / error, bouton « Nouveau message » après envoi.

3.8 Pages services dynamiques

  1. Hero immersif — image plein écran, dégradé sombre, badge icône, titre + sous-titre, lien retour.
  2. Bloc principal (2/3) — description longue, prestations incluses (coches colorées), processus 4 étapes (01 → 04).
  3. Sidebar sticky (1/3) — CTA noir avec accent du service, bouton appel, encart certifications.
  4. Bloc bas — 3 services complémentaires.

3.9 Pages légales

Layout partagé (components/LegalLayout.tsx) : lien retour, titre + date, contenu structuré (h2 + paragraphes), styles .legal-prose.

3.10 Page 404

Centrée, fond clair, code 404 géant, titre + description, bouton retour accueil.

04 Backend & API contact

4.1 Endpoint

POST /api/contact
Content-Type: application/json

4.2 Schéma de requête (Zod)

ChampTypeContraintes
namestringmin 2, max 120, requis
emailstringformat email valide, max 200, requis
phonestringmax 40, optionnel
companystringmax 160, optionnel
servicestringmax 80, optionnel
messagestringmin 20, max 5000, requis
websitestringmax 0 (honeypot anti-bot)

4.3 Codes de réponse

CodeCas
201Message reçu (et envoyé si SMTP configuré)
400Validation Zod en échec
429Rate-limit dépassé (5 req/h/IP)
502Erreur SMTP en aval
500Erreur interne inattendue

4.4 Mécanismes de sécurité

  • Validation stricte serveur via Zod — aucune confiance accordée au client.
  • Honeypot : champ website invisible — si rempli, requête rejetée silencieusement.
  • Rate-limit en mémoire : 5 requêtes par IP / heure (x-forwarded-for puis x-real-ip).
  • Échappement HTML de tous les contenus utilisateur dans le mail.
  • Runtime Node.js explicite — Nodemailer n'est pas Edge-compatible.
  • CONTACT_TO distinct de SMTP_USER, configurable par env.

4.5 Module lib/mailer.ts

  • sendContactEmail(payload) crée un Transporter nodemailer mis en cache.
  • Fallback gracieux : si les variables d'env manquent, retourne { sent: false, reason: 'smtp-not-configured' } — le formulaire reste fonctionnel en dev.
  • Email HTML stylisé : header dégradé bleu/vert, table des champs, message blanc, horodatage.
  • Plain text fallback pour clients sans HTML.
  • Reply-To = email du visiteur → réponse en un clic.

4.6 Configuration SMTP

SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=hl.human.link@gmail.com
SMTP_PASS=<app password 16 caractères>
CONTACT_TO=etatz56@gmail.com
Compatibilité fournisseurs : Gmail (avec App Password), OVH, Office 365, SendGrid, Mailgun. Port 465 → secure: true automatique ; port 587 ou autre → STARTTLS.

08 Performance & SEO

8.1 Performance

  • SSG sur toutes les pages publiques → HTML pré-rendu, TTFB minimal.
  • next/image avec sizes adaptés et priority sur les images critiques.
  • next/font Google Fonts via CSS (pas de FOIT/FOUT, swap).
  • Code splitting automatique par route.
  • Animations Framer Motion en viewport once pour ne pas re-rejouer.

8.2 SEO

  • metadata par page (title + description + OpenGraph) via Next.js metadata API.
  • Template de title : %s — GPE Énergies & Services.
  • metadataBase via NEXT_PUBLIC_SITE_URL.
  • 404 propre, HTML sémantique (h1 unique par page, <main>, <footer>), lang="fr".

8.3 Accessibilité

  • Contrastes WCAG AA (texte neutral-700+ sur blanc).
  • aria-label sur boutons icôniques (burger, sociaux).
  • Focus visible sur les inputs et boutons.
  • Navigation clavier fonctionnelle.

12 Recettes & validation

12.1 Tests fonctionnels manuels

  • Navigation entre toutes les pages (8 services + 3 légales + accueil).
  • Ancres internes scrollent à la bonne section.
  • Menu mobile s'ouvre/ferme proprement, scroll bloqué pendant l'ouverture.
  • Formulaire : succès, erreur validation, erreur réseau (couper internet).
  • Email reçu sur CONTACT_TO, format HTML correct.
  • Reply-To du mail = email saisi par le visiteur.
  • Honeypot : remplir website via DevTools → message rejeté silencieusement.
  • Rate-limit : 6e POST en moins d'1 h → réponse 429.

12.2 Tests responsive

  • iPhone SE (375 px) : tous les contenus lisibles, pas de débordement horizontal.
  • iPad portrait (768 px) : grille 2 colonnes services.
  • Desktop 1440 px : grille 4 colonnes services, hero ample.

12.3 Tests performance

  • Lighthouse mobile ≥ 90 sur Performance, Accessibility, Best Practices, SEO.
  • LCP < 2,5 s sur 4G simulé.
  • Aucune erreur console en production.
Validation finale — la recette est validée lorsque les trois familles de tests sont vertes et que l'envoi d'email arrive bien dans la boîte de réception de CONTACT_TO avec un Reply-To fonctionnel.
